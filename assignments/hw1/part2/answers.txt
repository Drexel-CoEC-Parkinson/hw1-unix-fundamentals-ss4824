# Q2a. A file has permissions rw-r----- and is owned by user alice and
# group mlteam. Can user bob, who is a member of mlteam, execute the
# file? Can he read it?

Q2a: Group perms are set to r--, so bob can only read.


# Q2b. You run ls -l and see -rwxrwxrwx on a file containing your API
# keys. Explain why this is a problem and what permissions you would
# set instead. Give the chmod command you would use.

Q2b: File with API keys probably doesn't need to be executable, so x perm can be absent for user, group, and other.
You wouldn't want random people using your API keys, so set other perms to ---.
Maybe you want to share API access to users in your group, so group perms can be r--, or even rw- if its ok for them to write.
Owner should be able to use manage their API keys freely, so user perms is rw-.

# Q2c. A directory has permissions rwxr-xr-x. You are not the owner
# and not in the group. Can you list the directory's contents? Can you
# create a new file inside it? Explain why or why not.

Q2c: Not owner and not in group means you are other. Other perms are r-x. 
Reading permission lets you list the directory contents. 
Lack of writing permission means you can't modify the folder or its contents.

# Q2d. What is the difference between chmod 755 and chmod u=rwx,go=rx?

Q2d: First command is in octal mode while second is in symbolic mode. 
Both achieve the same outcome: user having full permissions, group/other having read and execution permissions.
